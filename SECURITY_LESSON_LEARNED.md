## 📋 事件经过

### 错误操作

在首次上传技能到 Finance-calendar-bot 仓库时，**错误地将 `upload-to-github.sh` 脚本也上传了**。

### 问题分析

`upload-to-github.sh` 包含敏感信息：
- ❌ SSH 密钥路径：`~/.ssh/finance_calendar_key`
- ❌ 仓库 URL：`git@github.com:PorsenZ/Finance-calendar-bot.git`
- ❌ 上传逻辑和认证方式

**这些信息公开会导致**：
1. 攻击者可能知道 SSH 密钥位置
2. 知道目标仓库
3. 可能尝试暴力破解

### 根本原因

1. **安全意识不足**: 没有意识到上传脚本本身也是敏感信息
2. **检查不严格**: `.gitignore` 没有包含上传脚本
3. **匆忙操作**: 为了快速完成任务，没有仔细审查

## 🎯 安全意识提升

### 什么是敏感信息？

**明显敏感**：
- API Keys
- 密码
- SSH 私钥
- 数据库连接字符串

**容易被忽视的敏感信息**：
- SSH 密钥路径
- 内部仓库 URL
- 服务器 IP 地址
- 上传/部署脚本
- 配置文件模板（可能包含真实路径）

### 检查清单

每次上传前必须确认：

- [ ] .env 文件已排除
- [ ] 上传脚本已排除
- [ ] 没有硬编码的路径
- [ ] 没有内部 URL
- [ ] 没有服务器信息
- [ ] 使用了专用上传脚本
